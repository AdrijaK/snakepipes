HiC¶

Input requirements and outputs:¶

This pipeline requires paired-end reads fastq files as input in order to build a contact matrix and to call TADs. Prior to building a matrix, the pipeline maps reads against a user provided reference genome. The output of mapping step is then used for building the contact matrix.

Hi-C workflow v0.7.1 - MPI-IE workflow for Hi-C analysis

usage example:
HiC -i input-dir -o output-dir mm10

usage: HiC -i INDIR -o OUTDIR [-h] [-v] [-c CONFIGFILE]
           [--cluster_configfile CLUSTER_CONFIGFILE] [-j INT] [--local]
           [--snakemake_options STR] [--tempdir TEMPDIR] [--DAG]
           [--downsample INT] [--trim] [--trim_prg {cutadapt,trimgalore}]
           [--trim_options STR] [--fastqc] [--RF_resolution]
           [--enzyme {DpnII,HindIII}] [--bin_size INT] [--restrict_region STR]
           [--merge_samples] [--merge_bins NBINS_TOMERGE]
           [--findTADs_params STR] [--noTAD] [--noCorrect] [--distVsCount]
           [--distVsCount_params STR]
           GENOME

Positional Arguments¶

GENOME Genome acronym of target organism (supported: 'dm3', 'dm6', 'hs37d5', 'mm9', 'mm10', 'SchizoSPombe_ASM294v2')

required arguments¶

-i, --input-dir
 input directory containing the paired-end FASTQ files
-o, --output-dir
 output directory

general arguments¶

-v, --verbose verbose output (default: 'None')
-c, --configfile
 configuration file: config.yaml. In absence of a config file, the default options from the workflows would be selected from /workflows/<workflow_name>/defaults.yaml (default: 'None')
--cluster_configfile
 configuration file for cluster usage. In absence, the default options from shared/cluster.yaml and workflows/[workflow]/cluster.yaml would be selected (default: 'None')
-j, --jobs maximum number of concurrently submitted Slurm jobs / cores if workflow is run locally (default: 'None')
--local

run workflow locally; default: jobs are submitted to Slurm queue (default: 'False')

Default: False

--snakemake_options
 Snakemake options to be passed directly to snakemake, e.g. use --snakemake_options='--dryrun --rerun-incomplete --unlock --forceall'. (default: 'None')
--tempdir used prefix path for temporary directory created via mktemp. Created temp dir gets exported as $TMPDIR and is removed at the end of this wrapper! (default: 'None')
--DAG

If specified, a file called pipeline.pdf is produced in the output directory that shows the rules used and their relationship to each other.

Default: False

optional¶

--downsample downsample the given number of reads randomly from of each FASTQ file (default: 'None')
--trim Activate fastq read trimming. If activated, Illumina adaptors are trimmed by default. Additional parameters can be specified under --trim_options (default: 'None')
--trim_prg

Possible choices: cutadapt, trimgalore

trimming program: Cutadapt or TrimGalore (default: 'None')

--trim_options Additional option string for trimming program of choice. (default: 'None')
--fastqc run FastQC read quality control (default: 'None')
--RF_resolution
 Create Hi-C matrices at restriction fragment resolution. Using RF resolution would override the --bin_size argument. (default: 'None')
--enzyme

Possible choices: DpnII, HindIII

Which enzyme was used to create Hi-C library (default: 'None')

--bin_size Create Hi-C matrices at a given bin_size (default: 'None')
--restrict_region
 Restrict building of HiC Matrix to given region [Chr:Start-End]. Only one chromosome can also be specified (default: 'None')
--merge_samples
 Merge all HiC matrices and create a new matrix. (default: 'None')
--merge_bins If > 0 , create a lower resolution HiC matrix for each sample by merging the given number of bins. (default: 'None')
--findTADs_params
 parameters for HiCFindTADs. (default: 'None')
--noTAD Stop the pipeline before TAD calling. (default: 'None')
--noCorrect Stop the pipeline before correcting the matrix. (default: 'None')
--distVsCount Run hicDistVsCount. (default: 'None')
--distVsCount_params
 parameters to run hicDistVsCount. (default: 'None')

Structure of output directory¶

The HiC pipeline will generate output of the following structure:

$ tree -d -L 2 output-dir/

output-dir/
   |-- BWA
   |-- FASTQ
   |-- HiC_matrices
   |     |-- logs
   |     |-- QCplots
   |-- HiC_matrices_corrected
   |     |-- logs
   |-- TADs
         |--logs
